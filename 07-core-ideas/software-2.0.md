# Software 2.0 深度解读

> Karpathy 最重要的思想之一。原文章 2017 年发表在 Medium。

## 三岁版解释

**旧编程（Software 1.0）：**
你写一个菜谱，每一步都很清楚：
```
1. 拿鸡蛋
2. 打蛋
3. 放油
4. 炒
```
电脑严格按你的指令做。

**新编程（Software 2.0）：**
你不写菜谱了。你给 AI 看 100 万道做好的菜，AI 自己学会了怎么做菜。
你都不知道 AI 具体是怎么做的——但它做出来的菜很好吃。

**核心转变：从写指令 → 给例子**

## 为什么叫 Software 2.0？

Karpathy 认为，我们正在经历一个"软件范式转换"：

| Software 1.0 | Software 2.0 |
|---|---|
| 人写代码 | 人写数据 + 训练脚本 |
| 每条规则都明确 | 规则从数据中自动学出来 |
| 好理解但能力有限 | 难理解但能力无限 |
| 例如：if...else... | 例如：神经网络权重 |

**关键洞察：** 有些事情 1.0 根本解决不了（比如识别猫、理解语言），只有 2.0 能搞定。

## Software 2.0 的"开发"流程

```
1. 确定目标（你要 AI 学会什么）
2. 收集大量数据（给 AI 看例子）
3. 设计模型架构（选一种神经网络结构）
4. 训练（让 AI 自己学习）
5. 评估（检查 AI 学得好不好）
6. 部署（把 AI 放到实际使用中）
7. 重复 2-6 直到满意
```

**注意：** 开发者不再写"逻辑"，而是写"训练流程"和"数据管道"。

## Software 2.0 的栈

Karpathy 在 2018 年的演讲中把 Software 2.0 栈分成几层：

1. **硬件层** — GPU/TPU 等
2. **框架层** — PyTorch/TensorFlow 等
3. **模型层** — Transformer/BERT 等架构
4. **数据层** — 数据集、数据增强、合成数据
5. **训练层** — 优化器、损失函数、学习率调度
6. **评估层** — 基准测试、人工评估
7. **部署层** — 推理优化、边缘部署

**他的观点：** 未来大多数"编程"工作会集中在数据层和训练层，而不是传统的前端/后端开发。

## 在特斯拉的实践

在 Tesla，Karpathy 把 Software 2.0 用到了自动驾驶：

- **问题**：车需要识别路上的各种物体
- **1.0 方案**：写规则判断"什么是一个行人"（太难了）
- **2.0 方案**：给 AI 看 100 万张含行人的图片，AI 自己学会

**结果：**
- 神经网络处理所有视觉输入
- 在自研芯片上运行
- 通过 OTA 更新不断改进

## 实际影响

这个思想影响了：

1. **PyTorch/TensorFlow** 的普及——让更多人能做 Software 2.0
2. **数据标注行业**的兴起——数据变得比代码更重要
3. **AutoML**——让训练过程也自动化
4. **大语言模型**（ChatGPT 等）——最极致的 Software 2.0

## 批评和局限

Karpathy 也指出 Software 2.0 的缺点：

1. **不可解释**——你不知道 AI 为什么做某个决定
2. **需要大量数据**——不像 1.0 那样"写对了就行"
3. **难以调试**——改一行权重没有意义，需要重新训练
4. **黑盒风险**——在安全关键场景（如自动驾驶）中这是大问题

## 一句话总结

> **Software 2.0 不是写代码让电脑做事，而是给电脑例子让它自己学会做事。这是 AI 时代最重要的思维转变。**
