## Overview

The Course Project is an opportunity to apply what you have learned in class to a problem of your interest. Projects usually fall into two tracks:

1. **Applications:** Apply vision models to real-world problems related to your domain of interest (biology, engineering, physics, etc.).
2. **Models:** Build a new model or variant of existing models and apply it to vision tasks.

**Restriction:** The project must involve pixels of visual data in some form (this is a Computer Vision class). A pure NLP project is not appropriate.

## Collaboration

- Teams of up to **3** people.
- Projects with 3 people should have more impressive writeup and results.
- You may also work alone.

### Honor Code
- Consult any papers, books, online references, or publicly available implementations with proper citation.
- Do NOT look at another group's code or incorporate it.
- For combined projects with another class: obtain permission, clearly specify the unique CS231n portion, prepare separate reports.

### Generative AI Use
- Same guidelines as publicly available sources. All use must be explicitly documented.
- Using generative AI to write components of the final report is against the Honor Code.
- Generative AI may be used as an editing and formatting tool.

## Project Milestone Check-Ins

Three milestone check-ins. Each consists of a 10-minute discussion with course staff during TA office hours.

### Check-in Format
- Short presentation (2-5 slides), submitted to Gradescope prior to check-in
- Join QueueStatus for OH: https://queuestatus.com/queues/3447
- ~5 min presentation, remaining time for questions
- All group members expected to attend

### Milestone Schedule

#### Milestone 1 -- Problem + Related Work (May 15)
Focus: Well-defined problem grounded in prior work.
- Problem being solved, data plan, success criteria
- Evaluation method (metrics, baselines, benchmarks)
- Discussion of at least 3 relevant papers
- Suggested slides: problem motivation, dataset/evaluation, expected outcomes, related work

#### Milestone 2 -- Technical Approach (May 22)
Focus: Solidify method and implementation plan.
- Proposed approach, key design choices, intuitions
- Why method is expected to work vs. baselines
- Dataset usage, experimental setup, performance evaluation
- Suggested slides: method overview (diagrams), design intuition, baselines, experimental setup, implementation plan/progress

#### Milestone 3 -- Preliminary Results (May 29)
Focus: Results achieved so far.
- Initial results with quantitative evaluation (figures/tables)
- Comparison against baselines
- Analysis of what's working/not working
- Limitations and next steps
- Suggested slides: results, comparisons, analysis/insights, limitations, next steps

### Grading (per milestone)
- **Progress (1%):** Meaningful progress toward project goals
- **Clarity (1%):** Clear, logical presentation with well-structured slides
- **Robustness (1%):** Thoughtful reasoning, technical soundness, validation of approaches

## Final Report

- **Length:** 6-8 pages using provided template (CVPR-style format)
- **Structure:** Title/Author(s), Abstract (<300 words), Introduction (10%), Related Work (10%), Method/Approach, Experiments, Results, Discussion, Conclusion
- Reports will be posted online after class
- To opt out of posting, indicate via project registration form
- **Rubric:** Problem significance, technical soundness, experimental rigor, clarity of writing
